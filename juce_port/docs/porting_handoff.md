# Передача задачи по портированию MIDI генератора на JUCE

## Обзор задачи

Этот документ описывает передачу задачи по портированию MIDI генератора с Python на C++ с использованием фреймворка JUCE для создания VST плагина.

## Исходные данные

### Текущее приложение
- **Язык**: Python
- **Фреймворк GUI**: Dear PyGui
- **Архитектура**: Многопоточное приложение (GUI, Sequencer, MIDI)
- **Основные компоненты**:
  - Генераторы паттернов (Random, Euclidean, Dual Euclidean, Random V2)
  - Секвенсор с точностью 1мс
  - Музыкальная теория (14 ладов, система длительностей)
  - Looper (запись/воспроизведение паттернов)
  - Система пресетов

### Требования к портированию
- Создать VST3 плагин
- Сохранить всю функциональность оригинального приложения
- Обеспечить точность тайминга 1мс
- Поддерживать автоматизацию параметров в DAW
- Создать интуитивный пользовательский интерфейс

## Рекомендации по выполнению

### Этапы выполнения
1. **Подготовка**:
   - Установить Visual Studio с компилятором C++
   - Скачать и установить JUCE framework
   - Создать новый проект VST плагина в Projucer

2. **Портирование алгоритмов**:
   - Начать с одного генератора (например, RandomGenerator)
   - Постепенно добавлять остальные генераторы
   - Реализовать музыкальную теорию (Scales, Durations)

3. **Реализация секвенсора**:
   - Создать класс секвенсора в рамках VST архитектуры
   - Обеспечить точность таймера 1мс
   - Реализовать синхронизацию с внешним MIDI clock

4. **Looper функциональность**:
   - Реализовать запись паттернов
   - Создать систему воспроизведения
   - Настроить управление loop'ами

5. **MIDI обработка**:
   - Настроить отправку MIDI сообщений через VST хост
   - Реализовать прием и обработку MIDI clock
   - Настроить работу с различными MIDI каналами

6. **Пользовательский интерфейс**:
   - Создать UI компоненты в JUCE
   - Реализовать визуализацию паттернов
   - Настроить управление параметрами

7. **Система параметров и пресетов**:
   - Настроить параметры плагина для автоматизации в DAW
   - Реализовать систему сохранения/загрузки пресетов

8. **Тестирование и оптимизация**:
   - Тестирование в различных DAW
   - Проверка точности тайминга
   - Оптимизация производительности

### Приоритеты
1. **Функциональность** - корректная работа всех алгоритмов генерации
2. **Точность тайминга** - обеспечение 1мс точности
3. **Совместимость** - работа в различных DAW
4. **Производительность** - оптимизация для работы в real-time
5. **Пользовательский интерфейс** - интуитивность и удобство использования

### Критерии успеха
1. Плагин корректно работает в основных DAW (Ableton Live, FL Studio, Reaper, Cubase)
2. Точность тайминга составляет 1мс или лучше
3. Все алгоритмы генерации работают аналогично оригинальному приложению
4. Система параметров и пресетов полностью функциональна
5. Looper корректно записывает и воспроизводит паттерны
6. Плагин проходит тестирование на стабильность и отсутствие утечек памяти

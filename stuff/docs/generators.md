# Документация по Генераторам

Этот документ описывает логику и функции каждого музыкального генератора в системе. Все генераторы наследуются от базового класса `BaseGenerator` и реализуют метод `generate()`, который возвращает MIDI-события.

## 1. EuclideanGenerator (Евклидов Генератор)

**Назначение:**
Создает ритмические паттерны на основе евклидова алгоритма, который равномерно распределяет заданное количество импульсов (нот) по определенному количеству шагов. Это классический генератор для создания полиритмических и интересных грувов.

**Логика работы:**
1.  **Расчет паттерна:** Генератор использует функцию `_calculate_euclidean_pattern(steps, pulses)`, которая вычисляет бинарный паттерн (1 - есть нота, 0 - нет ноты).
2.  **Генерация нот:** На каждом шаге генератор проверяет, есть ли в паттерне "1". Если да, он создает MIDI-сообщение `note_on`.
3.  **Выбор ноты:**
    *   Если `deviation_range` равен 0, используется статическая нота (`note`).
    *   Если `deviation_range` > 0, генератор выбирает случайную ноту из текущей гаммы в диапазоне `[note - deviation_range, note + deviation_range]` (или `[note, note + deviation_range]`, если `deviation_is_bipolar` отключен).
4.  **Вероятность:** Параметр `note_probability` определяет вероятность срабатывания ноты на шаге, где она должна быть.

**Ключевые параметры:**
-   `steps`: Общее количество шагов в паттерне (например, 16 для 16-х нот).
-   `pulses`: Количество нот (импульсов), которые нужно распределить по шагам.
-   `note`: Базовая MIDI-нота для генерации.
-   `velocity`: Громкость нот.
-   `rate`: Скорость шага в долях такта (например, 0.25 для 16-х нот).
-   `deviation_range`: Диапазон для случайного выбора нот из гаммы.
-   `note_probability`: Вероятность (0.0 - 1.0) того, что нота будет сыграна.

## 2. DualEuclideanGenerator (Двойной Евклидов Генератор)

**Назначение:**
Объединяет два независимых евклидовых генератора (Машина А и Машина Б) для создания сложных полиритмов. Оба генератора работают параллельно от одного "мастер-шага".

**Логика работы:**
1.  **Два паттерна:** Генератор создает два независимых евклидовых паттерна (`pattern_a` и `pattern_b`) на основе параметров для каждой машины.
2.  **Общий шаг:** Генератор имеет общий счетчик шагов (`master_step`). На каждом шаге он проверяет наличие ноты в обоих паттернах по соответствующему индексу (`master_step % len(pattern)`).
3.  **Генерация нот:** Если в паттерне А на текущем шаге есть нота, генерируется событие для машины А. То же самое происходит для машины Б. Таким образом, на одном шаге могут быть сгенерированы две ноты (аккорд) или ни одной.
4.  **Выбор ноты:** Логика выбора ноты аналогична `EuclideanGenerator` для каждой из машин (А и Б) со своими параметрами `deviation_range`.

**Ключевые параметры:**
-   **Машина A:** `steps_a`, `pulses_a`, `note_a`, `velocity_a`, `deviation_range_a`.
-   **Машина B:** `steps_b`, `pulses_b`, `note_b`, `velocity_b`, `deviation_range_b`.
-   **Общие:** `rate`, `note_probability`.

## 3. RandomGenerator (Случайный Генератор)

**Назначение:**
Генерирует случайные ноты в заданном диапазоне и с учетом выбранной музыкальной гаммы. Полезен для создания мелодических линий или арпеджио.

**Логика работы:**
1.  **Выбор ноты:**
    *   Генератор получает список доступных нот из текущей гаммы (`scale_notes`).
    *   Этот список фильтруется по диапазону `min_note` и `max_note`.
    *   Из отфильтрованного списка выбирается случайная нота.
2.  **Расчет громкости:** Громкость (`velocity`) не просто случайна, а рассчитывается с использованием бета-распределения, что позволяет смещать ее к более тихим или громким значениям с помощью параметра `velocity_bias`.
3.  **Дополнительный CC:** Может генерировать дополнительное сообщение `Control Change` (CC 74), которое часто используется для управления фильтром синтезатора.

**Ключевые параметры:**
-   `min_note`, `max_note`: Диапазон MIDI-нот для генерации.
-   `max_velocity`: Максимальная громкость.
-   `velocity_bias`: Смещение громкости (0.0 - тише, 1.0 - громче).
-   `add_cc74`: Включить/выключить генерацию CC 74.
-   `note_probability`: Вероятность генерации ноты на каждом шаге.
-   `rate`: Скорость генерации нот.

## 4. RandomGeneratorV2 (Случайный Генератор v2)

**Назначение:**
"Эмбиентный" генератор, который создает более сложные и музыкально-осознанные случайные паттерны. Он чередует длинные одиночные ноты с быстрыми "вспышками" (bursts) нот.

**Логика работы:**
1.  **Циклы:** Генератор работает в циклах, длина которых определяется `base_duration`.
2.  **Режимы:** В начале каждого цикла он решает, будет ли это:
    *   **Одиночная нота:** С вероятностью `1 - burst_probability` генерируется одна длинная нота.
    *   **Вспышка (Burst):** С вероятностью `burst_probability` запускается серия коротких нот.
3.  **Логика вспышки:**
    *   "Вспышка" состоит из 8 шагов.
    *   Скорость нот во вспышке определяется `acceleration_strength`.
    *   Вероятность каждой ноты внутри вспышки задается паттерном `burst_pattern`.
4.  **Выбор ноты (Фильтрация баса):**
    *   Генератор определяет басовые ноты (тоника и квинта) в самой низкой октаве доступного диапазона.
    *   С вероятностью 50% выбирается одна из этих басовых нот, чтобы создать более стабильный гармонический фундамент. В противном случае выбирается любая нота из гаммы.

**Ключевые параметры:**
-   `min_note`, `max_note`: Диапазон нот.
-   `scale_name`, `root_note`: Название гаммы и ее тоника.
-   `base_duration`: Длительность одного цикла (например, 4.0 для целой ноты).
-   `burst_probability`: Вероятность возникновения "вспышки".
-   `burst_pattern`: Паттерн вероятностей для нот внутри вспышки.
-   `acceleration_strength`: Скорость нот во вспышке (например, 1/4 для 16-х).
-   `note_probability`: Общая вероятность генерации нот.

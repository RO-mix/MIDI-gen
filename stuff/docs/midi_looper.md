# Документация по MIDI-луперу (LooperModule)

Этот документ описывает внутреннюю логику и функции интеллектуального MIDI-лупера, реализованного в классе `LooperModule`.

## 1. Основная концепция

В отличие от простого лупера, который только записывает и воспроизводит MIDI, `LooperModule` представляет собой сложную систему, которая работает с музыкальными "сидами" (seeds) — короткими музыкальными фразами, захваченными у генераторов. Лупер анализирует эти сиды и может создавать на их основе вариации, а также предоставляет гибкие инструменты для манипуляции лупом.

Лупер оперирует **долями (beats)**, а не секундами, что обеспечивает точную синхронизацию с основным секвенсором.

## 2. Ключевые структуры данных

-   `loop`: Основной активный луп. Это список словарей, где каждый словарь представляет ноту:
    ```python
    {'pitch': 60, 'velocity': 100, 'start_beat': 0.0, 'end_beat': 0.5}
    ```
-   `pristine_loop`: "Нетронутая" копия лупа. Хранится для того, чтобы можно было отменить квантование (`unquantize_notes`) или сбросить вариации к исходному состоянию.
-   `loop_duration_beats`: Длительность лупа в долях.
-   `scale_notes`, `root_note`, `scale_degrees`: Музыкальный контекст (гамма, тоника), который используется для создания осмысленных вариаций.

## 3. Основные функции

### 3.1. Захват и запись

-   **`capture_seed(...)`**: Основной метод для "захвата" музыки.
    1.  Принимает на вход список нот от генератора, музыкальный контекст (гамму, темп) и длительность.
    2.  Нормализует ноты, чтобы луп начинался с 0-й доли.
    3.  Сохраняет результат в `loop` и создает копию в `pristine_loop`.
    4.  Запускает `_analyze_loop()` для анализа структуры лупа.

-   **`overdub_with_notes(...)`**: Наложение (overdubbing). Добавляет новые ноты поверх существующего лупа, не удаляя старые.

-   **`start_recording(...)` / `stop_recording()`**: Функции для записи MIDI в реальном времени (например, с MIDI-клавиатуры).
    -   `overdub=True`: режим наложения.
    -   `overdub=False`: режим замены (старый луп заменяется новым после записи).

### 3.2. Воспроизведение

-   **`play()` / `stop()`**: Управляют состоянием воспроизведения.
-   **`tick(master_beat)`**: "Сердце" лупера. Этот метод вызывается на каждом шаге главного секвенсора.
    1.  Рассчитывает текущую позицию внутри лупа (`current_loop_pos_beats`).
    2.  Определяет, какие ноты должны быть сыграны в промежутке времени между прошлым и текущим тиком.
    3.  Отправляет `note_on` сообщения для этих нот.
    4.  Планирует `note_off` сообщения для них (если не включен `pad_mode`).
-   **`pad_mode`**: Специальный режим, в котором `note_off` сообщения не отправляются автоматически. Это позволяет нотам звучать до тех пор, пока они не будут выключены вручную, что полезно для игры пэдами или дронами.

### 3.3. Манипуляции с лупом

-   **`generate_variations()`**: Создает "умные" вариации на основе текущего лупа и настроек `intensity`.
    -   **Bass variations:** Добавляет октавы вниз для басовых нот.
    -   **Mid variations:** Добавляет гармонические интервалы (терции, квинты) для нот в среднем диапазоне.
    -   **High variations:** Добавляет форшлаги (grace notes) к нотам в высоком диапазоне.

-   **`quantize_notes(grid_resolution_beats)`**: Квантует луп.
    -   Применяет квантование к `pristine_loop`, чтобы избежать накопления ошибок квантования.
    -   Выравнивает начальные позиции нот по заданной сетке (например, `0.25` для 16-х нот).
    -   Результат сохраняется в `loop`.

-   **`unquantize_notes()`**: Отменяет квантование, восстанавливая луп из `pristine_loop`.

-   **`double_loop()`**: Удваивает длину лупа, копируя все ноты и сдвигая их на `loop_duration_beats`.

-   **`split_loop(keep_first_half)`**: Разделяет луп пополам.
    -   `keep_first_half=True`: оставляет первую половину.
    -   `keep_first_half=False`: оставляет вторую половину, сдвигая ее к началу.

### 3.4. Управление и синхронизация

-   **`set_auto_recapture_period(period)`**: Устанавливает период (в количестве циклов лупа), через который лупер должен автоматически сделать новый захват сида у генератора. `0` отключает эту функцию.

-   **`_analyze_loop()`**: Внутренний метод, который анализирует луп и извлекает из него метаданные:
    -   Гистограмма высоты нот.
    -   Средняя длительность нот.
    -   Плотность нот.
    Эти данные могут использоваться для более сложных алгоритмов вариации в будущем.

## 4. Жизненный цикл лупа

1.  **Создание:** Луп создается через `capture_seed` или `start_recording`. В этот момент `loop` и `pristine_loop` идентичны.
2.  **Манипуляция:** Пользователь может применять к лупу различные функции:
    -   `quantize_notes` изменяет `loop`, но оставляет `pristine_loop` нетронутым.
    -   `generate_variations`, `double_loop`, `split_loop` изменяют `loop` и **обновляют** `pristine_loop`, так как эти изменения считаются новой "основой" для лупа.
3.  **Воспроизведение:** `tick()` постоянно проигрывает текущее состояние `loop`.
4.  **Сброс:** `unquantize_notes` возвращает `loop` к состоянию `pristine_loop`.
5.  **Пересоздание:** `capture_seed` или `stop_recording` (в режиме замены) полностью заменяют старый луп новым.

## 5. Пользовательский интерфейс (UI)

Этот раздел отвечает на вопросы о том, как пользователь взаимодействует с функциями лупера через графический интерфейс.

-   **Как запускаются основные функции (`capture_seed`, `generate_variations` и т.д.)?**
    Все основные действия привязаны к отдельным кнопкам:
    -   **`ЗАХВАТ`**: Запускает `capture_seed`. Рядом есть выпадающий список для выбора длительности захвата и чекбокс "ПОВЕРХ" для режима наложения.
    -   **`Вариация`**: Запускает `generate_variations`.
    -   **`x2`**: Запускает `double_loop`.
    -   **`/2`**: Запускает `split_loop`.

-   **Как устанавливается сетка квантования?**
    -   Для квантования используется кнопка **`Квантайз`**. Рядом с ней находится выпадающий список **`Сетка`**, в котором пользователь может выбрать разрешение сетки (например, "1/16", "1/8", "Выкл."). Выбор "Выкл." и нажатие на "Квантайз" отменяет квантование (`unquantize_notes`).

-   **Есть ли регулятор `intensity` для вариаций?**
    -   Да, в интерфейсе есть три отдельных слайдера для управления интенсивностью вариаций:
        -   **`Интенсивность Баса`**
        -   **`Интенсивность Середины`**
        -   **`Интенсивность Верха`**
    -   Каждый слайдер управляет соответствующим значением в словаре `intensity` лупера.

-   **Как включается и выключается `pad_mode`?**
    -   Для этого предназначен чекбокс с меткой **`PAD`**. Его включение и выключение напрямую вызывает метод `set_pad_mode()` лупера.

# Техническое задание: Random Generator v2.0
## Управление длительностями нот с системой ускорений

### Цель
Создать новую версию RandomGenerator с продвинутой системой управления длительностями нот, которая позволит создавать более музыкальные и динамичные паттерны через контролируемые ускорения.

### Текущее состояние
- Генератор использует `get_probalistic_duration()` для случайной длительности
- Нет контроля базовой длительности
- Нет возможности управлять ускорениями

### Новые функциональные требования

#### 1. Базовая длительность
**Параметр:** `base_duration` (float)
- Диапазон: 0.25 - 4.0 (16-я до целой ноты)
- Значения: 0.25=16-я, 0.5=8-я, 1.0=четверть, 2.0=половинная, 4.0=целая
- По умолчанию: 1.0 (четверть нота)

#### 2. Система ускорений
**Параметры:**
- `acceleration_prob` (float): вероятность ускорения (0.0-1.0)
- `acceleration_strength` (float): интенсивность ускорения (0.0-1.0)

**Механизм работы:**
- При `acceleration_strength = 0.0`: только базовая длительность
- При `acceleration_strength = 1.0`: максимальные ускорения
- Ускорения работают в 3 уровнях:
  - Уровень 1: базовая/2 (например, 8-я вместо 4-ой)
  - Уровень 2: базовая/4 (например, 16-я вместо 4-ой)
  - Уровень 3: базовая/8 (например, 32-я вместо 4-ой)

#### 3. Вероятностная матрица длительностей
**Реализация:**
```python
duration_weights = {
    'base': 1.0 - acceleration_prob,
    'level1': acceleration_prob * (1 - acceleration_strength) * 0.5,
    'level2': acceleration_prob * acceleration_strength * 0.3,
    'level3': acceleration_prob * acceleration_strength * 0.2
}
```

#### 4. Генерация длительности
**Алгоритм:**
1. Определить базовую длительность
2. Выбрать уровень ускорения по вероятностям
3. Рассчитать финальную длительность
4. Применить duration_bias для вариаций

# Техническое задание: Монофонический Random Generator v2.2

## Основные характеристики:
1. **Монофонический режим**:
   - Генератор воспроизводит только одну ноту одновременно
   - Новые ноты прерывают предыдущие (note-off перед note-on)
   - Параметр `monophonic: bool = True` (фиксирован)

2. **Ограничение ступеней лада**:
   - Нижний диапазон нот ограничивается 1, 3, 5, 6 ступенями выбранного лада
   - Новый параметр `scale_degrees: list = [0, 2, 4, 5]` (индексы ступеней)
   - Автоматическая фильтрация при изменении лада

## Доработки системы длительностей:
1. **Улучшенный алгоритм ускорений**:
```python
def get_duration():
    if random() < acceleration_prob:
        level = min(3, 1 + int(acceleration_strength * 3))
        return base_duration / (2 ** level)
    return base_duration
```

2. **Интеграция с монофонией**:
   - Гарантированный note-off перед новыми note-on
   - Корректная обработка legato-эффектов

## Пример кода для фильтрации ступеней:
```python
def get_bass_notes(scale_notes):
    return [scale_notes[i] for i in [0, 2, 4, 5] if i < len(scale_notes)]
```
